The system generates a Private Thread for every ticket.

  • Rule #1: Never reply in the public #mailbox channel.
  • Rule #2: Never ping the user outside the ticket.

This ensures 100% privacy and keeps public channels clean.

?note [text]
Internal log. Visible only to staff. Use for warnings or internal discussion.

?id
Retrieves the User ID. Mandatory for bans.

?close
Archives the session. Only use when resolution is confirmed.

Priority Level 1 (Critical)
Raids, Severe TOS violations, Doxxing. Requires immediate Admin escalation.

Priority Level 2 (Standard)
Harassment reports, Appeals, General inquiries.

Priority Level 3 (Low)
Role requests, Feedback.

Maintain neutrality. Do not use personal opinions.

Good: "We are reviewing your request."
Bad: "Idk let me ask someone lol."

Your goal is de-escalation, not argumentation.

Command: ?takeover

Hardware locks the thread to Admins only. Use only if a staff member is compromised or the situation involves legal threats.

If a user requests a Voice Chat (VC) for a ticket:

  • Always have a second staff member present (Two-Person Rule).
  • Do not record without consent.
  • Summarize the VC outcomes in the ticket using ?note afterwards.

All bans require proof. Use ?note to paste image links of evidence before closing a ticket.

If a user deletes a message, the bot logs it. Check the logs channel before replying.

Staff work is stressful. If a ticket becomes overwhelming:

  • Ask a colleague to take over.
  • Step away from the screen.
  • Do not engage if you are angry.
